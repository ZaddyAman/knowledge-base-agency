# Domain documentation

This is a single-context repository.

## Before exploring

- Read the root `CONTEXT.md`.
- Read relevant decisions under `docs/adr/` when that directory exists.
- If a file does not exist, proceed silently. Domain-modeling creates documentation lazily as decisions are resolved.

## Vocabulary

Use the canonical terms from `CONTEXT.md` in issue titles, specifications, module interfaces, tests, and user-facing language. Avoid synonyms that the glossary explicitly rejects.

If a required concept is absent or ambiguous, resolve it through domain-modeling before introducing a new term.

## ADR conflicts

Surface any conflict with an existing ADR explicitly. Do not silently override recorded decisions.
